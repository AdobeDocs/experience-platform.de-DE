# Wörterbuch der XDM-Felder


Das [!DNL Experience Data Model] (XDM) definiert einen Satz von Standardfeldern, die beim Erstellen von Adobe Experience Platform-Schemas verwendet werden können. Die Verwendung dieser Felder sichert ein konsistentes Verhalten aller [!DNL Platform]-Services, da alle Services, die XDM verwenden, die spezifischen Definitionen dieser Felder explizit verstehen.

Weitere Informationen zu den Bausteinen von Schemas, einschließlich Prinzipien und Best Practices beim Definieren von Schemas für [!DNL Experience Platform], finden Sie in den [Grundlagen der Schemakomposition](composition.md).

Die nachstehende Tabelle enthält den Feldnamen (nicht alle XDM-Felder werden mit &quot;xdm&quot;benannt) zusammen mit Titel und Beschreibung, Typ und Format, wo erforderlich, sowie den entsprechenden XDM-Typ und die Ressourcen (Schemas, Schemafeldgruppen und Datentypen), in denen das Feld zu finden ist.

Mithilfe der Suchfunktion im [!DNL Schema Editor] können Sie Details zu diesen Ressourcen anzeigen, wie im [Tutorial zum Schema-Editor](../tutorials/create-schema-ui.md#browse) gezeigt.

>[!NOTE]
>
> Wenn ein Feld mit **VERALTET** markiert ist, bedeutet dies, dass es in neuen Implementierungen nicht mehr verwendet werden sollte. Das Feld ist aus Gründen der Abwärtskompatibilität weiterhin verfügbar, wird jedoch in zukünftigen Versionen nicht mehr unterstützt.
