## Aktualisierungen der Benutzeroberfläche {#ux}

Adobe Experience Platform veröffentlicht Updates für die Domäne und Kopfzeilenleiste, um Ihr Erlebnis zu verbessern und andere [!DNL Experience Cloud]-Anwendungen zu vereinen.

- Einfacherer Wechsel zwischen Ihren Organisationen oder zu einer anderen Anwendung
- Verbessertes Handbuch, einschließlich spezieller Artikel und kontextbezogener Dokumentationen im Hilfemenü
- Möglichkeit, Feedback zu den [!DNL Experience Platform]- und Datei-Support-Tickets zu geben

Die Einführung des neuen Erlebnisses erfolgt schrittweise. Sie können das Erlebnis unter [https://experience.adobe.com/platform](https://experience.adobe.com/platform) Ansicht haben.

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] nutzt maschinelles Lernen und künstliche Intelligenz, um Erkenntnisse aus Ihren Daten zu entfesseln. In Adobe Experience Platform integriert hilft [!DNL Data Science Workspace] Ihnen bei der Erstellung von Prognosen mit Ihren Inhalten und Daten-Assets in allen Adoben. Eine der Möglichkeiten für [!DNL Data Science Workspace] ist die Verwendung von [!DNL JupyterLab]. [!DNL JupyterLab] ist eine Web-basierte Benutzeroberfläche für [[!DNL Project Jupyter]](https://jupyter.org/) und ist eng in Adobe Experience Platform integriert. Es bietet eine interaktive Entwicklungs-Umgebung für Datenwissenschaftler, mit [!DNL Jupyter]-Notebooks, -Code und -Daten zu arbeiten.

**Neue Funktionen**

| Funktion | Beschreibung |
|--- | ---|
| [!DNL JupyterLab Launcher] | Das [!DNL JupyterLab Launcher] enthält jetzt Starter für [!DNL Spark] 2.4 Notebooks. [!DNL Spark] 2.3 Notebook-Starter werden nun als veraltet markiert und in einer späteren Version entfernt. |
| [!DNL Spark] 2.4 | Neue Scala- ([!DNL Spark]) und PySpark-Rezepte verwenden jetzt [!DNL Spark] 2.4. |
| Kernels | Scala-Notebooks ([!DNL Spark]) werden jetzt über den Scala-Kernel verfasst. PySpark Notebooks werden jetzt über den [!DNL Python] Kernel verfasst. Der [!DNL Spark]- und PySpark-Kernel sind veraltet und sollen in einer folgenden Version entfernt werden. |
| Rezepte | Neue PySpark- und [!DNL Spark]-Rezepte folgen nun dem [!DNL Docker]-Arbeitsablauf ähnlich den Rezepten [!DNL Python] und R. |

Weitere Informationen zur Migration Ihrer Notebooks und Rezepte auf [!DNL Spark] 2.4 finden Sie im Handbuch [Notebook-Datenzugriff](../../data-science-workspace/jupyterlab/access-notebook-data.md). Allgemeine Informationen zum Data Science Workspace finden Sie in der [Übersichtsdokumentation](../../data-science-workspace/home.md).

## [!DNL Experience Platform Web SDK] und [!DNL Experience Platform Edge Network] {#edge}

Mit den Befehlen [!DNL Experience Platform Web SDK] und [!DNL Experience Platform Edge Network] können Benutzer Daten an Adobe Experience Platform und andere Adobe Solutions in Echtzeit für Endbenutzergeräte und Browser senden. Die aktuellste Liste von Anwendungsfällen finden Sie in unserem [öffentlichen Fahrplan](https://github.com/adobe/alloy/projects/5), der häufig aktualisiert wird.

**Neue Funktionen**

| Funktion | Beschreibung |
|--- | ---|
| ECID-Unterstützung | Das SDK unterstützt standardmäßig ECID ohne zusätzliche Bibliotheken oder zu installierende Informationen. |
| Konfigurationsoberfläche | Verwalten Sie Ihre Konfigurations-ID-Einstellungen mit der neuen Benutzeroberfläche für die Edge-Konfiguration in [!DNL Launch]. Sie müssen sich auf der Zulassungsliste befinden, um Zugriff zu erhalten. |
| Adobe Experience Platform Web SDK Mixin | Eine Mischung, die mit dem [!DNL Experience Platform web SDK] verwendet wird und alle unterstützten Felder umfasst. |
| Einwilligungseinstellungen des Kurses | Gibt Firmen die Kontrolle über die Teilnahme und Abmeldung von [!DNL Experience Platform Web SDK]. |
| Clientseitige Debugging-Unterstützung im neuen [!DNL Experience Cloud Debugger Extension] | Sehen Sie sich Anforderungen von [!DNL Experience Platform web SDK] sowie Edge Traces an, um zu sehen, wie Daten durch das System fließen. |
| Adobe Analytics | Senden Sie Daten über die Edge-Konfiguration an [!DNL Analytics] Report Suites. XDM wird in Kontextdaten reduziert und unterstützt Multi-Suite-Tagging. |
| Adobe Target | Unterstützung für Adobe Target. Einschließlich VEC, Form-Based Composer, A/B, XT, Automated Personalization, MVT. |
| Adobe Audience Manager Support | Unterstützung für ID-Syncs, URL-Ziele und Cookie-Ziele.[!DNL Audience Manager] |
| Identitätssynchronisierung | Umbenannt in `setCustomersIds` in `syncIdentity`, um es klarer zu gestalten. |
| XDM Object Builder | In der Starterweiterung können Sie jetzt XDM-Objekte als Datenelemente erstellen. |

Weitere Informationen zu [!DNL Platform Web SDK] und [!DNL Edge Network] finden Sie in der [Dokumentation](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform ermöglicht die Bereitstellung koordinierter, konsistenter und relevanter Erlebnisse für Kunden, unabhängig davon, wo und wann diese mit Ihrer Marke interagieren. Mit [!DNL Real-time Customer Profile] können Sie eine ganzheitliche Ansicht jedes einzelnen Kunden sehen, die Daten aus mehreren Kanälen, einschließlich Online-, Offline-, CRM- und Drittanbieterdaten, kombiniert. [!DNL Profile] ermöglicht es Ihnen, Ihre unterschiedlichen Kundendaten zu einer einheitlichen Ansicht zusammenzuführen, die Ihnen einen umsetzbaren Zeitstempel für jede Kundeninteraktion bietet.

**Neue Funktionen**

| Funktion | Beschreibung |
| -----------| ---------- |
| Neue Profil-Exportmetriken | Es wurden Metriken zu Exportaufträgen für Profile hinzugefügt, die die Gesamtzahl der exportierten Profil und die Anzahl der Profil in jedem Namensraum anzeigen. |
| Neue Metriken zur Beobachtbarkeit | Observability Insights API verfügt jetzt über die folgenden Metriken für die Streaming-Erfassung nach [!DNL Profile]: Eingehende Anforderungsrate, Erfolgreiche Ingestion-Rate, Aufgenommene Datensatzgröße. |
| Massenendpunkt des GET | Der API [!DNL Real-time Customer Profile] wurde ein Massen-GET-Endpunkt hinzugefügt, um das Abrufen mehrerer Ergebnisse in einem einzigen API-Aufruf zu ermöglichen. Für Segmentdefinitionen, Segmentaufträge und Zusammenführungsrichtlinien können Sie jetzt GET bis zu 100 IDs stapeln. |
| Profil nach Identität durchsuchen | In der Benutzeroberfläche [!DNL Platform] können Sie jetzt einen Identitäts-Namensraum auswählen und einen Identitätswert angeben, um ein Profil zu durchsuchen. |

## Quellen {#sources}

Adobe Experience Platform kann Daten aus externen Quellen erfassen, während Sie diese Daten mithilfe der [!DNL Platform]-Dienste strukturieren, beschriften und erweitern können. Daten können aus verschiedenen Quellen erfasst werden, z. B. aus Adobe-Anwendungen, Cloud-basiertem Speicher, Software von Drittanbietern und Ihrem CRM-System.

[!DNL Experience Platform]Im Rahmen von stehen eine RESTful-API und interaktive Benutzeroberfläche zur Verfügung, mit deren Hilfe Sie auf unkomplizierte Weise Verbindungen zu Datenquellen verschiedener Anbieter einrichten können. Mit diesen Quellverbindungen können Sie sich authentifizieren und eine Verbindung zu externen Datenspeichern und CRM-Diensten herstellen, Zeiten für Erfassungsläufe festlegen und den Durchsatz der Datenerfassung verwalten.

**Neue Funktionen**

| Funktion | Beschreibung |
| ------- | ----------- |
| Zusätzliche API- und UI-Unterstützung für Cloud-Datenspeicherung | Neue Quell-Connectoren für [!DNL Azure File Storage]. |
| Zusätzliche API- und UI-Unterstützung für Datenbanken | Neue Quellschnittstellen für [!DNL Azure Data Explorer], IBM DB2 und [!DNL Oracle DB]. |
| Datenfreigabe für Adobe Audience Manager bis [!DNL Experience Platform] | Der Bereitstellungsprozess für den [!DNL Audience Manager] Connector wurde aktualisiert. [!DNL Audience Manager] Datensätze für  [!DNL Real-time Customer Profile] sind jetzt standardmäßig deaktiviert. Sie können manuell festlegen, welche Datensätze für [!DNL Profile] hochgeladen werden sollen. Die neuen Standardeinstellungen sind nicht rückwirkend und wirken sich nur auf die Bereitstellung neuer [!DNL Audience Manager]-Connectors aus. Weitere Informationen finden Sie im Benutzerhandbuch [Datensätze](../../catalog/datasets/user-guide.md). |
