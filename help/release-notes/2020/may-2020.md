## Aktualisierungen der Benutzeroberfläche {#ux}

Adobe Experience Platform veröffentlicht Aktualisierungen für die Domäne und die Kopfzeilenleiste, um Ihr Erlebnis zu verbessern und mit anderen [!DNL Experience Cloud]-Anwendungen zu vereinheitlichen.

- Einfacherer Wechsel zwischen Ihren Organisationen oder zu einer anderen Anwendung
- Verbessertes Handbuch, einschließlich spezieller Artikel und kontextbezogener Dokumentationen im Hilfemenü
- Möglichkeit, Feedback zu den [!DNL Experience Platform]- und Support-Tickets einzureichen

Die Einführung des neuen Erlebnisses erfolgt allmählich. Sie können das Erlebnis unter [https://experience.adobe.com/platform](https://experience.adobe.com/platform) anzeigen.

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] nutzt maschinelles Lernen und künstliche Intelligenz, um Erkenntnisse aus Ihren Daten zu gewinnen. [!DNL Data Science Workspace] ist in Adobe Experience Platform integriert und hilft Ihnen bei der Erstellung von Prognosen auf der Basis Ihrer Inhalts- und Datenelemente in allen Adobe-Lösungen. Eine der Möglichkeiten, wie [!DNL Data Science Workspace] dies erreicht, besteht in der Verwendung von [!DNL JupyterLab]. [!DNL JupyterLab] ist eine Web-basierte Benutzeroberfläche für [[!DNL Project Jupyter]](https://jupyter.org/) und ist eng in Adobe Experience Platform integriert. Es bietet eine interaktive Entwicklungsumgebung, in der Datenwissenschaftler mit [!DNL Jupyter] Notebooks, Code und Daten arbeiten können.

**Neue Funktionen**

| Funktion | Beschreibung |
|--- | ---|
| [!DNL JupyterLab Launcher] | Das [!DNL JupyterLab Launcher] enthält jetzt Starter für [!DNL Spark] 2.4 Notebooks. [!DNL Spark] 2.3 Notebook-Starter werden jetzt als veraltet markiert und in einer nachfolgenden Version entfernt. |
| [!DNL Spark] 2,4 | Neue Scala-Rezepte ([!DNL Spark]) und PySpark-Rezepte verwenden jetzt [!DNL Spark] 2.4. |
| Kernels | Scala ([!DNL Spark]) Notebooks werden jetzt über den Scala-Kernel erstellt. PySpark-Notebooks werden jetzt über den Kernel [!DNL Python] erstellt. Der [!DNL Spark]- und PySpark-Kernel sind veraltet und werden in einer nachfolgenden Version entfernt. |
| Rezepte | Neue PySpark- und [!DNL Spark]-Rezepte folgen nun dem [!DNL Docker]-Workflow ähnlich den [!DNL Python]- und R-Rezepten. |

Weitere Informationen zur Migration Ihrer Notebooks und Rezepte zur Verwendung von [!DNL Spark] 2.4 finden Sie im [Handbuch zum Datenzugriff für Notebooks](../../data-science-workspace/jupyterlab/access-notebook-data.md). Allgemeine Informationen zu Data Science Workspace finden Sie in der [Übersichtsdokumentation](../../data-science-workspace/home.md).

**[!DNL Facebook]**

Weitere Informationen zur neuen Funktion finden Sie auf der Seite [Facebook-Ziel](../../destinations/catalog/social/facebook.md) .

## [!DNL Experience Platform Web SDK] und [!DNL Experience Platform Edge Network] {#edge}

Die [!DNL Experience Platform Web SDK] und [!DNL Experience Platform Edge Network] ermöglichen es Benutzern, Daten an Adobe Experience Platform und andere Adobe-Lösungen in Echtzeit für Endbenutzergeräte und -browser zu senden. Die aktuelle Liste der Anwendungsfälle finden Sie in unserer [öffentlichen Roadmap](https://github.com/adobe/alloy/projects/5) , die häufig aktualisiert wird.

**Neue Funktionen**

| Funktion | Beschreibung |
|--- | ---|
| Unterstützung für ECID | Das SDK unterstützt standardmäßig ECID ohne zusätzliche zu installierende Bibliotheken oder Informationen. |
| Konfigurationsoberfläche | Verwalten Sie Ihre Konfigurations-ID-Einstellungen mit der neuen Edge-Konfigurationsoberfläche in [!DNL Launch]. Sie müssen sich auf der Zulassungsliste befinden, um darauf zugreifen zu können. |
| Adobe Experience Platform Web SDK-Schemafeldgruppe | Eine Feldergruppe zur Verwendung mit [!DNL Experience Platform web SDK] , die alle unterstützten Felder umfasst. |
| Einverständniskontrollen für Kurse | Ermöglicht Unternehmen die Kontrolle über das Opt-in und Opt-out von [!DNL Experience Platform Web SDK]. |
| Clientseitige Debugging-Unterstützung in der neuen [!DNL Experience Cloud Debugger Extension] | Sehen Sie sich Anforderungen von [!DNL Experience Platform web SDK] sowie Edge-Traces an, um zu sehen, wie Daten durch das System fließen. |
| Adobe Analytics | Senden Sie Daten über die Edge-Konfiguration an [!DNL Analytics] Report Suites. XDM wird in Kontextdaten reduziert und unterstützt Multi-Suite-Tagging. |
| Adobe Target | Unterstützung für Adobe Target. Einschließlich VEC, Form-Based Composer, A/B, XT, Automated Personalization, MVT. |
| Adobe Audience Manager-Support | Unterstützung für [!DNL Audience Manager] ID-Synchronisierungen, URL-Ziele und Cookie-Ziele. |
| Identitätssynchronisierung | `setCustomersIds` wurde in `syncIdentity` umbenannt, um es deutlicher zu machen. |
| XDM Object Builder | In der Launch-Erweiterung können Sie jetzt XDM-Objekte als Datenelemente erstellen. |

Weitere Informationen zu [!DNL Platform Web SDK] und [!DNL Edge Network] finden Sie in der [Dokumentation](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform ermöglicht die Bereitstellung koordinierter, konsistenter und relevanter Erlebnisse für Kunden, unabhängig davon, wo und wann diese mit Ihrer Marke interagieren. Mit [!DNL Real-time Customer Profile] können Sie eine ganzheitliche Ansicht jedes einzelnen Kunden anzeigen, die Daten aus mehreren Kanälen kombiniert, einschließlich Online-, Offline-, CRM- und Drittanbieter-Daten. [!DNL Profile] ermöglicht es Ihnen, Ihre unterschiedlichen Kundendaten in einer einheitlichen Ansicht zusammenzufassen, die eine umsetzbare, mit Zeitstempel versehene Übersicht über jede Kundeninteraktion bietet.

**Neue Funktionen**

| Funktion | Beschreibung |
| -----------| ---------- |
| Neue Profilexport-Metriken | Es wurden Metriken zu Profilexportvorgängen hinzugefügt, die die Gesamtzahl der exportierten Profile und die Anzahl der Profile in jedem Namespace anzeigen. |
| Neue Metriken zu Observability Insights | Die Observability Insights-API verfügt jetzt über die folgenden Metriken für die Streaming-Erfassung in [!DNL Profile]: Eingehende Anforderungsrate, Erfolgreiche Aufnahmerate, aufgenommene Datensatzgröße. |
| Massen-GET-Endpunkt | Bulk-GET-Endpunkt zur [!DNL Real-time Customer Profile]-API hinzugefügt, um das Abrufen mehrerer Ergebnisse in einem einzelnen API-Aufruf zu ermöglichen. Sie können jetzt GET bis zu 100 IDs für Segmentdefinitionen, Segmentaufträge und Zusammenführungsrichtlinien stapeln. |
| Profil nach Identität durchsuchen | In der [!DNL Platform]-Benutzeroberfläche können Sie jetzt einen Identitäts-Namespace auswählen und einen Identitätswert angeben, um ein Profil zu durchsuchen. |

**Fehlerkorrekturen**

## Quellen {#sources}

Adobe Experience Platform kann Daten aus externen Quellen erfassen und gleichzeitig diese Daten mithilfe von [!DNL Platform]-Diensten strukturieren, beschriften und erweitern. Daten können aus verschiedenen Quellen erfasst werden, z. B. aus Adobe-Anwendungen, Cloud-basiertem Speicher, Software von Drittanbietern und Ihrem CRM-System.

[!DNL Experience Platform]Im Rahmen von stehen eine RESTful-API und interaktive Benutzeroberfläche zur Verfügung, mit deren Hilfe Sie auf unkomplizierte Weise Verbindungen zu Datenquellen verschiedener Anbieter einrichten können. Mit diesen Quellverbindungen können Sie sich authentifizieren und eine Verbindung zu externen Datenspeichern und CRM-Diensten herstellen, Zeiten für Erfassungsläufe festlegen und den Durchsatz der Datenerfassung verwalten.

**Neue Funktionen**

| Funktion | Beschreibung |
| ------- | ----------- |
| Zusätzliche API- und UI-Unterstützung für Cloud-Speichersysteme | Neue Quell-Connectoren für [!DNL Azure File Storage]. |
| Zusätzliche API- und UI-Unterstützung für Datenbanken | Neue Quell-Connectoren für [!DNL Azure Data Explorer], IBM DB2 und [!DNL Oracle DB]. |
| Datenfreigabe zwischen Adobe Audience Manager und [!DNL Experience Platform] | Der Bereitstellungsprozess für den Connector [!DNL Audience Manager] wurde aktualisiert. [!DNL Audience Manager] -Datensätze für  [!DNL Real-time Customer Profile] sind jetzt standardmäßig deaktiviert. Sie können manuell auswählen, welche Datensätze an [!DNL Profile] weitergeleitet werden sollen. Die neuen Standardeinstellungen sind nicht rückwirkend und wirken sich nur auf die Bereitstellung neuer [!DNL Audience Manager] Connectoren aus. Weitere Informationen finden Sie im [Benutzerhandbuch zu Datensätzen](../../catalog/datasets/user-guide.md). |

**Bekannte Probleme**

- Keine.
