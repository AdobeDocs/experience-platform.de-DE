## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] nutzt maschinelles Lernen und künstliche Intelligenz, um Erkenntnisse aus Ihren Daten zu entfesseln. Integrated into Adobe Experience Platform, [!DNL Data Science Workspace] helps you make predictions using your content and data assets across Adobe solutions. Eine der Möglichkeiten [!DNL Data Science Workspace] dafür ist der Einsatz von [!DNL JupyterLab]. [!DNL JupyterLab] ist eine Web-basierte Benutzeroberfläche für [[!DNL Project Jupyter]](https://jupyter.org/) und ist eng in Adobe Experience Platform integriert. It provides an interactive development environment for data scientists to work with [!DNL Jupyter] notebooks, code, and data.

**Neue Funktionen**

| Funktion | Beschreibung |
|--- | ---|
| [!DNL JupyterLab Launcher] | Das [!DNL JupyterLab Launcher] enthält nun Starter für [!DNL Spark] 2,4 Notebooks. [!DNL Spark] 2.3 Notebook-Starter werden nun als veraltet markiert und in einer späteren Version entfernt. |
| [!DNL Spark] 2.4 | Neue Scala- ([!DNL Spark]) und PySpark-Rezepte verwenden jetzt [!DNL Spark] 2.4. |
| Kernels | Scala ([!DNL Spark]) Notebooks werden jetzt über den Scala Kernel verfasst. PySpark Notebooks werden jetzt über den [!DNL Python] Kernel verfasst. Der Kernel [!DNL Spark] und der PySpark sind veraltet und sollen in einer folgenden Version entfernt werden. |
| Rezepte | Neue PySpark- und [!DNL Spark] Rezepte folgen jetzt dem [!DNL Docker] Arbeitsablauf ähnlich wie [!DNL Python] und R-Rezepte. |

Weitere Informationen zur Migration Ihrer Notebooks und Rezepte auf [!DNL Spark] 2.4 finden Sie im Handbuch [für den Datenzugriff](../../data-science-workspace/jupyterlab/access-notebook-data.md)auf Notebooks. Weitere allgemeine Informationen zum Data Science Workspace finden Sie in der [Übersichtsdokumentation](../../data-science-workspace/home.md).

## [!DNL Experience Platform Web SDK] und [!DNL Experience Platform Edge Network] {#edge}

Das [!DNL Experience Platform Web SDK] und [!DNL Experience Platform Edge Network] ermöglichen Benutzern das Senden von Daten an Adobe Experience Platform und andere Adobe Solutions in Echtzeit für Endbenutzergeräte und Browser. Die aktuellste Liste der Anwendungsfälle finden Sie in unserem häufig aktualisierten [Fahrplan](https://github.com/adobe/alloy/projects/5) .

**Neue Funktionen**

| Funktion | Beschreibung |
|--- | ---|
| ECID-Unterstützung | Das SDK unterstützt standardmäßig ECID ohne zusätzliche Bibliotheken oder zu installierende Informationen. |
| Konfigurationsoberfläche | Verwalten Sie Ihre Konfigurations-ID-Einstellungen mit der neuen Benutzeroberfläche für die Edge-Konfiguration in [!DNL Launch], müssen Sie sich auf der Zulassungsliste befinden, um Zugriff zu erhalten. |
| Adobe Experience Platform Web SDK Mixin | Ein Mixin, das alle unterstützten Felder [!DNL Experience Platform web SDK] umfasst. |
| Einwilligungseinstellungen des Kurses | Bietet Firmen Kontrolle über die Teilnahme und die Abmeldung von [!DNL Experience Platform Web SDK]Anmeldedaten. |
| Clientseitige Debugging-Unterstützung in der neuen [!DNL Experience Cloud Debugger Extension] | Sehen Sie sich Anforderungen von [!DNL Experience Platform web SDK] sowie Edge Traces an, um zu sehen, wie Daten durch das System fließen. |
| Adobe Analytics | Senden Sie Daten über die Edge-Konfiguration an [!DNL Analytics] Report Suites. XDM wird in Kontextdaten reduziert und unterstützt Multi-Suite-Tagging. |
| Adobe Target | Unterstützung für Adobe Target. Einschließlich VEC, Form-Based Composer, A/B, XT, Automated Personalization, MVT. |
| Adobe Audience Manager Support | Unterstützung für [!DNL Audience Manager] ID-Synchronisierungen, URL-Ziele und Cookie-Ziele. |
| Identitätssynchronisierung | Umbenannt `setCustomersIds` in `syncIdentity` klarer zu gestalten. |
| XDM Object Builder | In der Starterweiterung können Sie jetzt XDM-Objekte als Datenelemente erstellen. |

For more information on [!DNL Platform Web SDK] and [!DNL Edge Network], refer to the [documentation](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform ermöglicht die Bereitstellung koordinierter, konsistenter und relevanter Erlebnisse für Kunden, unabhängig davon, wo und wann diese mit Ihrer Marke interagieren. With [!DNL Real-time Customer Profile], you can see a holistic view of each individual customer that combines data from multiple channels, including online, offline, CRM, and third party data. [!DNL Profile] ermöglicht es Ihnen, Ihre unterschiedlichen Kundendaten zu einer einheitlichen Ansicht zusammenzuführen, die Ihnen einen umsetzbaren Zeitstempel für jede Kundeninteraktion bietet.

**Neue Funktionen**

| Funktion | Beschreibung |
| -----------| ---------- |
| Neue Profil-Exportmetriken | Es wurden Metriken zu Exportaufträgen für Profile hinzugefügt, die die Gesamtzahl der exportierten Profil und die Anzahl der Profil in jedem Namensraum anzeigen. |
| Neue Metriken zur Beobachtbarkeit | Die Insight-API für die Observability Insights-Funktion verfügt jetzt über die folgenden Metriken für die Streaming-Erfassung in [!DNL Profile]: Eingehende Anforderungsrate, Erfolgreiche Ingestion-Rate, Aufgenommene Datensatzgröße. |
| Massenendpunkt des GET | Der [!DNL Real-time Customer Profile] API wurde ein Massen-GET-Endpunkt hinzugefügt, um das Abrufen mehrerer Ergebnisse in einem einzigen API-Aufruf zu ermöglichen. Für Segmentdefinitionen, Segmentaufträge und Zusammenführungsrichtlinien können Sie jetzt GET bis zu 100 IDs stapeln. |
| Profil nach Identität durchsuchen | In der [!DNL Platform] Benutzeroberfläche können Sie jetzt einen Identitäts-Namensraum auswählen und einen Identitätswert angeben, um ein Profil zu durchsuchen. |

## Quellen {#sources}

Adobe Experience Platform can ingest data from external sources while allowing you to structure, label, and enhance that data using [!DNL Platform] services. Daten können aus verschiedenen Quellen erfasst werden, z. B. aus Adobe-Anwendungen, Cloud-basiertem Speicher, Software von Drittanbietern und Ihrem CRM-System.

[!DNL Experience Platform]Im Rahmen von stehen eine RESTful-API und interaktive Benutzeroberfläche zur Verfügung, mit deren Hilfe Sie auf unkomplizierte Weise Verbindungen zu Datenquellen verschiedener Anbieter einrichten können. Mit diesen Quellverbindungen können Sie sich authentifizieren und eine Verbindung zu externen Datenspeichern und CRM-Diensten herstellen, Zeiten für Erfassungsläufe festlegen und den Durchsatz der Datenerfassung verwalten.

**Neue Funktionen**

| Funktion | Beschreibung |
| ------- | ----------- |
| Zusätzliche API- und UI-Unterstützung für Cloud-Datenspeicherung | Neue Quell-Connectoren für [!DNL Azure File Storage]. |
| Zusätzliche API- und UI-Unterstützung für Datenbanken | Neue Quellschnittstellen für [!DNL Azure Data Explorer], IBM DB2 und [!DNL Oracle DB]. |
| Adobe Audience Manager für die [!DNL Experience Platform] Datenfreigabe | Der Bereitstellungsprozess für den [!DNL Audience Manager] Connector wurde aktualisiert. [!DNL Audience Manager] Datensätze für [!DNL Real-time Customer Profile] sind jetzt standardmäßig deaktiviert. Sie können manuell festlegen, für welche Datensätze Werbung gemacht werden soll [!DNL Profile]. Die neuen Standardeinstellungen sind nicht rückwirkend und wirken sich nur auf die Bereitstellung neuer [!DNL Audience Manager] Connectors aus. Weitere Informationen finden Sie im [Benutzerhandbuch](../../catalog/datasets/user-guide.md)für Datasets. |
